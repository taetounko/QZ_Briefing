# 아키텍처

## 현재 구조

```text
QZ_Briefing/
├── config/            설정 파일 위치
├── data/              수집·저장 데이터 위치
├── logs/              실행 로그 위치
├── reports/           생성된 브리핑 위치
├── src/qz_briefing/   애플리케이션 소스 패키지
└── tests/             테스트 패키지
```

## 설계 원칙

- 데이터 수집값, 계산값, 해석문을 서로 분리한다.
- 계좌 정보는 읽기 전용 경계를 유지한다.
- 주문 및 자동매매 경로는 두지 않는다.
- 브리핑에는 기준시각과 데이터 누락 상태를 함께 전달한다.
- 작은 기능 단위로 구현하고 각 기능 구현 후 테스트한다.

## 향후 구성 경계

세부 모듈과 데이터 흐름은 `TASKS.md`의 데이터 모델 설계 단계에서 정의한다. 현재 단계에서는 실제 분석 엔진이나 OpenAPI 연결 구조를 확정하거나 구현하지 않는다.
