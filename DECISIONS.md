# 결정 기록

## D-001 프로젝트 목적

- 국내 증시 브리핑 전용 프로그램으로 개발한다.
- 자동매매와 주문 실행 기능은 범위에서 제외한다.

## D-002 실행 환경

- Windows PowerShell 네이티브 환경을 사용한다.
- Python 3.11 32-bit 가상환경 `.venv`를 사용한다.
- WSL은 사용하지 않는다.

## D-003 초기 단계 범위

- 현재 단계에서는 문서, 패키지, 테스트 및 데이터 디렉터리 골격만 생성한다.
- 실제 분석 엔진과 키움 OpenAPI+ 연결 코드는 이후 작업으로 남긴다.
- 외부 패키지는 필요가 확인되기 전까지 추가하지 않는다.

## D-004 코드 출처

- 기존 QZ_Squeeze 코드나 다른 프로젝트의 코드를 검색·복사·재사용하지 않는다.

## D-005 데이터와 출력

- 수집값, 계산값, 해석문은 분리한다.
- 모든 브리핑에 데이터 기준시각과 누락 여부를 표시한다.
- 브리핑 해석은 현재 의미와 주의사항을 포함한다.

## D-006 PyQt5 및 QAxWidget 환경

- Python 3.11.9 32-bit를 사용한다.
- GUI 및 ActiveX 환경에 PyQt5를 사용한다.
- Windows 네이티브 환경을 사용한다.
- 빈 `QAxWidget` 객체 생성까지 환경 검증을 완료했다.
- 주문 기능은 포함하지 않는다.

## D-007 키움 OpenAPI+ OCX 연결상태 검사

- 키움 OpenAPI+ 컨트롤 식별자로 `KHOPENAPI.KHOpenAPICtrl.1`을 사용한다.
- 실제 로그인 전에 `GetConnectState`로 연결상태를 먼저 검사한다.
- 연결상태 `0`은 환경오류가 아니라 미접속 상태로 구분한다.
- 이번 단계에서는 `CommConnect`를 호출하지 않는다.

## D-008 키움 OpenAPI+ 로그인 연결 이벤트 검사

- 실행 시 `GetConnectState`를 먼저 확인한다.
- 연결상태가 `0`일 때만 `CommConnect`를 한 번 호출한다.
- 로그인 대기에는 별도의 `QEventLoop`를 사용한다.
- QApplication 전체 종료와 로그인 이벤트 루프 종료를 분리한다.
- `OnEventConnect` 이벤트로 로그인 성공 여부를 확정한다.
- `OnEventConnect` 성공 후 `GetConnectState`가 `1`인지 다시 확인한다.
- 인증정보는 키움 로그인창에서만 입력하고 프로그램에 저장하지 않는다.
- 현재 단계에서는 반복 재접속을 구현하지 않는다.

## D-009 장시간 연결 관리

- 무한 재접속을 금지한다.
- 기본 최대 재접속 횟수는 3회다.
- 기본 재접속 지연은 60초다.
- 실제 시간과 연결 객체를 주입 가능하게 설계한다.
- 계정정보는 연결 관리자에 저장하지 않는다.

## D-010 키움 QAx 어댑터 경계

- QApplication은 어댑터 내부에서 생성하지 않고 상위 런타임이 관리한다.
- OCX 호출과 재접속 정책을 각각 어댑터와 연결 관리자로 분리한다.
- `OnEventConnect` 오류코드는 등록된 리스너 방식으로 전달한다.
- 어댑터는 계정정보나 인증정보를 저장하지 않는다.
