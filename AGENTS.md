# QZ Briefing 작업 규칙

이 저장소에서 작업하는 모든 사람과 에이전트는 다음 규칙을 준수한다.

- 주문 관련 OpenAPI 함수는 구현하지 않는다.
- 계좌 정보는 읽기 전용으로만 사용한다.
- 실제 보유종목 데이터와 개인정보를 Git에 커밋하지 않는다.
- 기존 QZ_Squeeze 코드를 재사용하지 않는다.
- 한 번에 하나의 작은 기능만 구현한다.
- 기능 구현 후 반드시 테스트를 실행한다.
- 실패한 테스트를 숨기거나 삭제하지 않는다.
- 데이터 수집값, 계산값, 해석문을 서로 분리한다.
- 브리핑 지표는 수치만 출력하지 않고 현재 의미와 주의사항을 간결하게 설명한다.
- 브리핑 문구 기본 형식은 `해석 2~3문장 / 현재 해석 / 주의`이다.
- 사용자가 명시하지 않은 매매 규칙을 임의로 만들지 않는다.
- 장전 최종 브리핑은 09:00에 실행하고 08:00~09:00 장전시장 변화를 감시한다.
- 예상체결·호가 불균형을 실제 투자자 수급으로 표현하지 않는다.
- 연도별 휴장일 하드코딩을 필수 운영 조건으로 만들지 않고 실제 Kiwoom 시장 상태를 우선한다.
- 07:30 자동 실행은 바탕화면 아이콘 미배포 기간의 운영 보조 수단이다.
- 무인 로그인은 Kiwoom OpenAPI+ 공식 자동로그인만 사용하며 인증정보 저장·자동입력을 구현하지 않는다.
- 손실률만으로 물타기 후보를 만들지 않는다.
- 판단에는 근거·확인 조건·무효 조건·행동 수준을 포함한다.
- 예상체결과 실제 체결 수급을 명확히 구분한다.
- 신뢰도가 낮으면 단정적인 표현을 사용하지 않는다.
- 매수·매도 확정 지시 대신 조건부 판단을 제공한다.
- 배포형과 바탕화면 아이콘은 전체 기능 완성 후 진행한다.
- 실행 중 Windows 자동 절전을 방지하고 종료 시 반드시 해제한다.
- heartbeat와 일일 운영 요약에는 민감정보를 저장하지 않는다.
- 예정 브리핑 누락은 하루 최대 한 번만 자동 복구한다.
- watchdog과 기존 timeout 복구를 중복 실행하지 않는다.
- 07:30 작업은 WakeToRun을 사용한다.
- 원격 알림 실패가 브리핑 생성을 중단하지 않게 한다.
- Telegram Token은 DPAPI로 암호화하고 Git·로그·JSON에 평문 저장하지 않는다.
- 원격 메시지에 실제 계좌번호와 인증정보를 포함하지 않는다.
- 브리핑은 저장 성공 후에만 원격 전송한다.
- 동일 브리핑을 중복 전송하지 않는다.
- Telegram 네트워크 처리는 Qt·Kiwoom 이벤트루프를 차단하지 않는다.
- 실제 코드 작업과 검증은 에이전트가 직접 수행하고 사용자에게 명령 복사를 요청하지 않는다.
- 기능별로 분석, 구현, 테스트, 자체검수, 재테스트, 커밋, push 순서를 지킨다.
- 관련 없는 변경을 초기화하거나 삭제하지 않으며 `git reset --hard`, `git clean`을 사용하지 않는다.
- 실제 Kiwoom 로그인이나 운영체제 승인창이 필요한 경우에만 사용자 승인을 요청한다.
- PowerShell 환경을 보고할 때 일반 PowerShell과 `(.venv) PS D:\QZ_Briefing>`을 구분한다.
- 장마감 브리핑은 15:40에 실행한다.
- 장마감 결과는 다음 거래일 장전 브리핑에 연결한다.
- 직전 거래일은 단순 전날이 아니라 최신 유효 장마감 결과로 찾는다.
- 배포형·바탕화면 아이콘 작업은 핵심 기능 완성 후 진행한다.
- UI는 기존 파이프라인 결과와 저장파일만 표시하며 직접 TR을 호출하지 않는다.
- 창을 닫으면 트레이에서 계속 실행한다.
- 종료는 트레이 종료 또는 20:00 graceful shutdown으로만 수행한다.
- 바탕화면 아이콘과 배포형 설치는 전체 기능 완성 후 진행한다.
- 일시적 연결 끊김은 상태 재확인 후 자동 재연결한다.
- 예정 브리핑은 연결 복구 후 중복 없이 실행한다.
- 정규 브리핑은 일시적 연결·TR 오류에 최대 1회 자동 재시도한다.
- Ctrl+C는 traceback 없이 graceful shutdown한다.
